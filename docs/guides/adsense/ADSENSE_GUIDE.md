# 구글 애드센스 광고 삽입 가이드

## 📌 개요
RE: Report Effortless 서비스에 구글 애드센스 광고를 삽입하는 방법

---

## 1단계: 애드센스 계정 준비

### 1.1 계정 생성
1. https://www.google.com/adsense 접속
2. Google 계정으로 로그인
3. 사이트 URL 입력 (예: https://yourusername.github.io/RE)
4. 국가 선택 및 약관 동의
5. 결제 정보 입력

### 1.2 사이트 인증 코드 삽입
애드센스에서 제공하는 코드를 모든 페이지의 `<head>` 태그에 삽입:

```html
<script async src="https://pagead2.googlesyndication.com/pagead/js/adsbygoogle.js?client=ca-pub-XXXXXXXXXX"
     crossorigin="anonymous"></script>
```

### 1.3 필수 페이지 추가
- **개인정보 보호정책** (Privacy Policy) 페이지 필수
- **연락처** (Contact) 페이지 권장
- **이용약관** (Terms of Service) 권장

### 1.4 승인 대기
- 승인까지 보통 **1-2주** 소요
- 승인 기준:
  - 고유하고 가치 있는 콘텐츠
  - 충분한 콘텐츠량 (최소 20-30개 페이지)
  - 사용자 친화적인 네비게이션
  - 명확한 개인정보 보호정책

---

## 2단계: 광고 단위 생성

승인 후 애드센스 대시보드에서:

1. **광고 > 광고 단위 기준** 클릭
2. 광고 유형 선택:
   - **디스플레이 광고**: 가장 일반적, 크기 자동 조정
   - **인피드 광고**: 콘텐츠 목록 사이에 삽입
   - **기사 내 광고**: 긴 텍스트 콘텐츠 내부
3. 광고 이름 지정 (예: "사이드바 광고", "Footer 광고")
4. 크기 및 스타일 설정
5. **코드 받기** 클릭

---

## 3단계: 광고 배치 위치 추천

### 📍 권장 광고 배치 위치

#### 1. **index.html** (메인 페이지)
- 위치: 기능 카드 아래, Footer 위
- 광고 유형: 디스플레이 광고 (Responsive)
- 크기: 전체 너비 (Auto)

#### 2. **report.html** (보고서 작성 페이지)
- 위치: 입력 영역 하단 또는 우측 사이드바
- 광고 유형: 디스플레이 광고 (300x250 또는 Responsive)
- 주의: 사용자 경험을 방해하지 않도록 신중히 배치

#### 3. **settings.html** (설정 페이지)
- 위치: Footer 버튼 영역 하단
- 광고 유형: 디스플레이 광고 (Responsive)

#### 4. **guide 페이지들** (가이드 문서)
- 위치: 각 섹션 사이 또는 페이지 하단
- 광고 유형: 인피드 광고 또는 디스플레이 광고

#### 5. **notice.html, changelog.html, donate.html**
- 위치: 콘텐츠 하단, 홈으로 돌아가기 버튼 위
- 광고 유형: 디스플레이 광고 (Responsive)

## 4단계: 광고 코드 예시

### 기본 디스플레이 광고 코드:
```html
<div class="ad-container">
    <ins class="adsbygoogle"
         style="display:block"
         data-ad-client="ca-pub-XXXXXXXXXX"
         data-ad-slot="1234567890"
         data-ad-format="auto"
         data-full-width-responsive="true"></ins>
    <script>
         (adsbygoogle = window.adsbygoogle || []).push({});
    </script>
</div>
```

### CSS 스타일 (광고 컨테이너):
```css
.ad-container {
    max-width: 1200px;
    width: 100%;
    margin: 40px auto;
    padding: 20px;
    text-align: center;
    background: var(--bg-secondary);
    border: 1px solid var(--border-color);
    border-radius: 12px;
}

.ad-container::before {
    content: 'Advertisement';
    display: block;
    font-size: 0.75em;
    color: var(--text-tertiary);
    margin-bottom: 10px;
    text-transform: uppercase;
    letter-spacing: 0.5px;
}
```

## 5단계: 광고 성과 최적화 팁

### ✅ DO (권장)
- **적절한 위치**: 콘텐츠와 자연스럽게 어우러지는 위치
- **반응형 디자인**: 모바일에서도 잘 보이도록
- **가독성 유지**: 광고가 콘텐츠를 가리지 않도록
- **로딩 속도**: 광고로 인해 페이지 속도 저하 방지
- **사용자 경험 우선**: 광고보다 사용자 경험이 중요

### ❌ DON'T (금지)
- **과도한 광고**: 페이지당 3-4개 이상 배치 금지
- **클릭 유도**: "여기를 클릭하세요" 등의 문구 금지
- **가짜 콘텐츠**: 광고를 콘텐츠처럼 위장 금지
- **중요 버튼 근처**: "보고서 작성" 같은 주요 버튼 근처 배치 금지
- **입력 영역 방해**: 사용자가 입력하는 영역 근처 배치 금지

## 6단계: 광고 배치 실제 구현

제가 각 페이지에 광고를 삽입할 준비가 되었습니다. 
다음 정보를 제공해주시면 즉시 적용할 수 있습니다:

1. **애드센스 클라이언트 ID**: `ca-pub-XXXXXXXXXX`
2. **광고 슬롯 ID**: 각 광고 단위별로 다름 (예: `1234567890`)
3. **원하는 광고 배치 위치**: 위의 추천 위치 중 선택

## 💡 대안: 다른 광고 플랫폼

애드센스 승인이 어려운 경우 대안:

3. **Ezoic**: AI 기반 광고 최적화
4. **직접 광고**: 스폰서십이나 제휴 마케팅

## ❓ 자주 묻는 질문

**Q: GitHub Pages에서 애드센스를 사용할 수 있나요?**
A: 네, 가능합니다. 단, 커스텀 도메인이 있으면 승인이 더 쉽습니다.

**Q: 승인이 거부되면 어떻게 하나요?**
A: 콘텐츠를 보강하고 개인정보 보호정책을 추가한 후 재신청하세요.

**Q: 광고가 페이지 속도를 느리게 하나요?**
A: 애드센스는 비동기 로딩을 사용하므로 영향이 적습니다. 단, 과도한 광고는 피하세요.

**Q: 광고 수익은 언제 받을 수 있나요?**
A: $100 이상 누적되면 다음 달에 지급됩니다.
