🔒 보안 & 개인정보

RE를 안전하게 사용하는 방법을 알아봅니다 (모든 사용자 필독!)

⭐ 필독 권장

⚠️ 이 페이지는 꼭 읽어주세요!

RE는 상담 내용을 다루는 만큼 개인정보 보호가 매우 중요합니다. 이 페이지에서는 데이터가 어떻게 처리되는지, 무엇을 주의해야 하는지 투명하게 설명합니다.

📊 데이터는 어디에 저장되나요?

RE는 여러분의 데이터를 다른 곳에 저장합니다. 무엇이 어디에 저장되는지 정확히 알려드립니다.

항목 저장 위치 인터넷 전송 설명
🔑 API 키 💻 사용자 컴퓨터 ❌ 없음 브라우저의 저장공간(LocalStorage)에 암호화되어 저장
📝 상담 메모 💻 사용자 컴퓨터 ❌ 없음 브라우저에만 저장, 외부로 전송되지 않음
📋 작성 히스토리 💻 사용자 컴퓨터 ❌ 없음 최대 50개까지 로컬(사용자 컴퓨터)에만 저장
⚙️ 설정값 💻 사용자 컴퓨터 ❌ 없음 스타일 설정 등 브라우저에만 저장
📄 보고서 생성 🌐 AI 서버 ✅ 임시 전송 생성 시에만 임시로 전송되고, 처리 후 즉시 삭제
💬 AI 챗봇 🌐 AI 서버 ✅ 임시 전송 수정 요청 시에만 임시 전송 후 즉시 삭제

💡 용어 설명

  • 💻 사용자 컴퓨터: 여러분이 지금 사용 중인 컴퓨터의 브라우저 저장공간 (LocalStorage)
  • 🌐 AI 서버: 인터넷 너머에 있는 AI 프로그램이 실행되는 컴퓨터 (Groq, OpenRouter)
  • 임시 전송: 처리를 위해 잠깐 보냈다가 결과를 받으면 즉시 삭제됨
  • LocalStorage: 브라우저가 제공하는 저장공간으로, 해당 컴퓨터에서만 접근 가능

🔒 왜 이 툴은 안전할 수밖에 없나요?

🎯 핵심 원리: "중앙 서버가 없다"

RE는 네이버, 카카오, 구글처럼 중앙 서버에 데이터를 모으지 않습니다. 대신 여러분의 컴퓨터에서 직접 AI 서비스(Groq/OpenRouter)를 호출하는 방식입니다.

💡 실생활 비유로 이해하기

❌ 일반 서비스 (위험할 수 있음)

비유: 식당에 가서 주문하기
→ 주문 내역이 식당 컴퓨터에 저장됨
→ 식당이 망하거나 해킹당하면 내 정보도 유출

예: 네이버, 카카오 등 대부분의 서비스
✓ 편리하지만 데이터가 계속 남음
✗ 회사가 해킹당하면 내 데이터도 위험

✅ RE 방식 (안전)

비유: 집에서 직접 배달앱으로 주문하기
→ 주문 정보가 바로 식당으로만 전달
→ 식당은 음식만 만들고 주문서를 버림

RE는 중간 서버 없이 직접 AI에 연결
✓ RE 서버에는 아무것도 저장 안됨
✓ AI는 처리 후 즉시 데이터 삭제

🔐 3가지 안전 장치

1️⃣
RE 서버는 데이터를 보지도, 저장하지도 못함
여러분 컴퓨터 → (RE 웹사이트는 그냥 통과) → AI 서버
마치 편지를 우체부가 읽지 않고 배달만 하는 것과 같습니다.
2️⃣
AI 서버도 데이터를 저장하지 않음
Groq, OpenRouter는 GDPR(유럽 개인정보법) 준수 기업
API 방식은 처리 후 즉시 삭제가 법적으로 보장됩니다.
3️⃣
여러분만 데이터를 소유하고 통제함
히스토리, 메모는 100% 여러분 컴퓨터에만 저장
브라우저 데이터 삭제 = 모든 기록 완전 제거

✅ 결론: 물리적으로 저장할 곳이 없음

RE는 여러분의 데이터를 저장할 서버 자체가 없습니다.
AI는 계산기처럼 "계산만 하고 결과만 돌려주는" 역할입니다.
따라서 해킹이나 유출이 구조적으로 불가능합니다.

⚠️ 그래도 조심해야 할 것

  • API 키 보호: 은행 비밀번호처럼 절대 공유 금지
  • 공용 PC: 브라우저에 데이터가 남을 수 있음 (시크릿 모드 권장)
  • 개인정보 최소화: 실명보다는 "A씨", "내담자" 사용

🔐 보안 체크리스트

✅ 안전하게 사용하기
🔑
API 키 보호: API 키는 절대 다른 사람과 공유하지 마세요. 누출되면 즉시 Groq/OpenRouter 콘솔에서 키를 삭제하고 새로 발급받으세요.
🏢
공용 PC 주의: 공용 컴퓨터 사용 후에는 반드시 브라우저 캐시와 쿠키를 삭제하세요. 또는 시크릿 모드(Incognito)로 사용하세요.
🔍
개인정보 최소화: 보고서에 실명, 주민번호, 연락처 등은 최대한 쓰지 마세요. "A씨", "내담자" 같은 익명 표현을 사용하세요.
💾
정기 백업: 중요한 보고서는 "내보내기" 기능으로 정기적으로 백업하세요. USB나 클라우드에 안전하게 보관하세요.
🌐
안전한 네트워크: 카페나 공공 Wi-Fi보다는 집이나 직장의 안전한 네트워크에서 사용하세요.

❓ 보안 관련 자주 묻는 질문

Q1. AI 서버에서 정말 즉시 삭제되나요?

네! Groq와 OpenRouter 모두 API 호출 완료 후 데이터를 즉시 삭제한다고 공식적으로 명시하고 있습니다. 이들은 GDPR, HIPAA 등 국제 개인정보 보호 규정을 준수합니다.

Q2. API 키가 유출되면 어떻게 되나요?

누군가 여러분의 API 키를 알게 되면:

  • 💰 Groq: 무료 한도를 소진할 수 있음 (금전적 손실은 없음)
  • 💰 OpenRouter: 여러분 계정으로 AI 사용료가 청구됨 (금전적 손실 발생)

해결: 즉시 해당 서비스 콘솔에서 키를 삭제하고 새로 발급받으세요.

Q3. 회사/기관 보안 정책에 위배되지 않나요?

이는 각 기관의 정책에 따라 다릅니다. 사용 전에:

  • 소속 기관의 개인정보 보호 담당자에게 문의
  • RE: 의 데이터 처리 방식 설명 (이 페이지 참고)
  • 필요시 기관 내부 승인 절차 진행

Q4. 브라우저 데이터를 삭제하면?

브라우저 캐시/쿠키를 삭제하면 히스토리와 설정이 모두 사라집니다. 중요한 데이터는 반드시 "내보내기"로 백업하세요.

📋 권장사항 vs 금지사항

✅ 이렇게 하세요

  • 익명화된 표현 사용 ("내담자 A")
  • 개인 컴퓨터에서 사용
  • 정기적 백업
  • 안전한 네트워크 사용
  • API 키 안전하게 보관

❌ 이런 것은 피하세요

  • 실명, 주민번호, 연락처 기재
  • 공용 PC 무분별 사용
  • API 키 타인 공유
  • 공공 Wi-Fi에서 사용

🎯 요약: 핵심만 기억하세요

✅ 안전한 점

  • 대부분의 데이터는 사용자 컴퓨터에만 저장됩니다
  • AI 서버로 전송되는 데이터는 즉시 삭제됩니다
  • 모든 통신은 HTTPS 암호화로 보호됩니다
  • 데이터는 AI 학습에 사용되지 않습니다

⚠️ 주의할 점

  • 보고서 생성 시 메모가 AI 서버로 임시 전송됩니다
  • 개인식별정보는 최소화하세요 (이름, 주민번호, 연락처 등)
  • Beta Test 단계로 기능 및 보완이 완벽히 구현되어 있지 않으니, 개인정보에 유의하세요
  • 중요한 데이터는 별도로 백업하세요
